13. 설정

13.5. 시스템 운영 현황

13.5.1. Open-POP 점검

Open-POP Agent 점검 및 스케줄러 작업 수행내역과 수행결과를 확인할 수 있는 화면이다.

[Agent 점검] 탭

Open-POP DB, Open-POP Agent(Master, Collect, Gather), 관리대상 DB의 Instance 상태에 대한 점검을 수행한다.

Agent 점검 수행

화면 우측 [점검 수행] 버튼을 클릭한다. 점검 결과 Agent가 정상일 경우 녹색, 오류가 있을 경우 빨강색으로 조회된다.

참고 사항

  • - Agent 상태는 default.properties에 정의한다.(키워드는 일치해야 한다.)
  • - openpop_arch_ver=2
  • - agent_timeout=5
  • - masterip=112.24.102.154(고객사별 설정)
  • - masterport=1987(고객사별 설정)
  • - gather_server_ip=181.119.103.34(openpop_arch_ver=2일 경우, 고객사별 설정)
  • - gather_server_port=5001(openpop_arch_ver=2일 경우, 고객사별 설정)

[Scheduler 점검] 탭

스케줄러 작업 수행 내역과 수행결과를 확인할 수 있는 화면이다.

Scheduler 점검 수행

Scheduler 점검 탭을 클릭하면 스케줄러 점검이 자동 수행된다. 점검 결과 정상일 경우 녹색, 오류가 있을 경우 빨강색, 미수행일 경우 회색으로 조회된다.
