8. 품질 점검

8.1. SQL 표준 점검

8.1.1. SQL 표준 점검 작업

[SQL 표준 품질점검 작업] 탭

개발자 작성한 SQL을 적재하여 SQL표준에 대한 품질을 점검한다. 품질점검을 수행하기 위해서는 기준정보가 사전 등록되어 있어야 하며 이에 대한 설명은 FAQ를 참조한다.

SQL 적재

SQL 표준 품질검토를 진행할 SQL을 일괄 업로드 한다.
적재 파일 포맷은 가급적 *.txt나 *.sql을 사용하기를 권장하고 SQL은 DB에서 실행할 수 있는 형태여야 한다.
단위 SQL의 끝은 세미콜론(;)으로 끝나야 한다.

SQL적재방법은 아래와 같다.

  1. - 프로젝트를 선택한 후 [SQL적재] 버튼 클릭하면 SQL적재 팝업 창이 뜸
  2. - SQL적재 팝업에서 [Choose File] 버튼을 클릭하여 적재하고자 하는 파일을 선택
  3. - 팝업에서 [SQL적재] 버튼 클릭

적재가 완료되면 “SQL 적재가 완료되었습니다. 적재된 SQL 수:[적재건수]”가 메시지 창에 뜨고, [Ok] 버튼 클릭시 검색 영역의 ‘SQL 수’에 적재된 SQL 수가 조회된다.

품질검토작업

SQL 표준에 대한 품질검토를 수행하고 작업이 완료되면 수행 결과에 대한 집계 정보를 제공한다. 품질검토작업을 수행하면 “품질검토작업을 수행하시겠습니까?”라는 메시지 창이 뜬다. [YES] 버튼을 선택할 경우 금일 수행된 데이터를 삭제하고, 새로운 데이터를 생성한다.

품질검토작업이 수행 중일 경우 “선택한 프로젝트에서 진행 중인 품질검토 작업이 있습니다. 품질검토 작업을 다시 진행하려면 [강제완료처리] 버튼을 클릭하여 강제 완료 처리 후 수행 바랍니다!”라는 메시지 창이 뜬다. 재수행 할 경우 금일 수행된 데이터를 삭제하고, 새로운 데이터를 생성한다.

품질검토작업시 현재 작업상태를 알기 위해서는 [검색] 버튼을 클릭하면, 검색영역의 ‘작업상태’에서 현재 작업상태를 확인할 수 있다.

‘작업상태’의 의미는 아래와 같다.

  • - 미검토: 당일 품질검토 작업이 한번도 수행되지 않음
  • - 검토중: 현재 품질검토를 진행 중인 상태(1/3~3/3- 임시 테이블1작업~임시 테이블 3작업)
  • - 검토완료: 당일 최종 품질검토작업이 정상적으로 완료함
  • - 강제완료: 당일 최종 품질검토작업이 강제완료 처리됨

강제완료처리

품질검토 작업 시 진행 중인 작업을 중단해야 할 사유가 발생하면 [강제완료처리] 버튼을 클릭한다. 클릭하면 품질검토작업 시에 사용하는 임시 테이블의 정보를 모두 삭제하여 재수행 할 수 있다.

엑셀

현재 데이터를 엑셀형식으로 다운로드하여 보여준다.

[SQL 품질점검 집계표] 탭

SQL 표준 품질점검 수행 결과에 대한 집계 정보 및 미준수 SQL의 상세한 정보를 제공한다.

엑셀

집계 데이터와 표준 미준수 SQL 데이터를 엑셀형식으로 다운로드하여 보여준다.

[SQL 품질점검 예외 대상 관리] 탭

품질점검 결과에 대하여 예외로 처리되어야 할 대상을 관리(조회/등록/수정/삭제)한다.

엑셀

현재 데이터를 엑셀형식으로 다운로드하여 보여준다.

일괄 업로드

다수의 데이터를 일괄 업로드 한다.
업로드 후 성공 건수, 실패 건수, 실패 행에 대해 보여준다.

예제

엑셀 업로드가 완료되었습니다. 총 [ 12 ] 건 중 성공건수 : [ 10 ]건, 실패 행 : [5, 11] 번째 행
마지막으로 오류가 발생한 원인
[품질 점검 지표 코드]는 필수로 입력되어야 하며, [업무 코드],[디렉토리명],[SQL 식별자(DBIO)] 등 이 셋 중의 하나는 입력되어야 합니다.

데이터 등록

[초기화] 버튼을 누른 후 데이터를 순서대로 입력 한다.

데이터 수정

원하는 데이터 클릭 후 수정한다.
수정 시 음영 영역은 데이터 수정이 불가하다.

데이터 삭제

원하는 데이터를 클릭 후 삭제한다.

템플릿 다운로드

데이터를 업로드 할 수 있는 양식을 다운받는다.
