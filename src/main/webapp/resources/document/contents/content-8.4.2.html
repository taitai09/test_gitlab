8.4. SQL 성능 자동 점검

8.4.2. 성능 자동 점검 대상 관리

SQL 자동 성능 점검을 수행할 대상을 관리(추가, 수정, 삭제)한다. 성능 점검 대상 범위는 특정 OWNER, 특정 테이블들, 특정 MODULE들을 등록할 수 있으며 성능 점검 시 등록된 대상 범위에 대해서만 성능 점검을 수행한다.

관리 항목의 의미는 아래와 같다.

  • - 프로젝트명 : [필수] 선정 대상은 상단 프로젝트에서 선택된 프로젝트명을 보여준다.
  • - DB : [필수] 관리가 가능한 DB 목록을 보여준다.
  • - 성능점검대상범위 : [필수] 대상 범위를 지정한다. OWNER를 선택하면 OWNER 항목만 선택 가능, 테이블명 선택하면 OWNER와 테이블명 항목이 입력 가능, MODULE을 선택하면 MODULE 항목만 입력 가능
  • - OWNER : [선택] DB 항목 선택에 따라서 관리할 OWNER 목록을 보여준다.
  • - 테이블명 : [선택] 관리할 테이블명을 입력한다.
  • - MODULE : [선택] 관리할 MODULE 정보를 입력한다.

테이블 일괄 등록/삭제

다수의 테이블을 등록/삭제할 경우 편의성을 위해 일괄 등록/삭제 기능을 제공한다. SQL 성능 자동 점검 대상 관리 화면에서 [테이블 일괄 등록/삭제] 버튼을 클릭하면 다음과 같은 팝업 화면을 보여준다.

선택된 DB 정보와 OWNER 정보에 따른 테이블 목록과 대상 테이블 목록을 보여준다. 테이블 목록과 대상 테이블 목록은 다중 선택이 가능하며, 등록/삭제 버튼을 통해서 테이블 목록에서 대상 테이블 목록 또는 대상 테이블 목록에서 테이블 목록으로 이동한다.
