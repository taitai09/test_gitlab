11.1. SQL 성능 진단

11.1.3. Unknown SQL 장애 예측

갑작스런 DB 자원 사용량 증가 원인을 분석하기 위하여 정규 프로그램에서 수행되는 SQL과 개발자 또는 비정규 프로그램을 통해 수행되는 Adhoc SQL의 사용량을 비교하여 장애 원인을 다각적으로 신속하게 예측 분석하기 위한 정보를 제공한다.

Unknown SQL 장애 예측 조회

DB / INSTANCE / 분석일 등의 조건을 선택하여 검색하면 Unknown SQL에 의한 장애 원인 예측 정보가 조회된다. Unknown SQL의 Resource 사용량이 많은 경우 TOP SQL 및 TOP 모듈을 통해 Resource 를 많이 사용하는 SQL과 모듈을 확인하여 장애 원인을 분석한다.

장애 원인 예측을 위해 제공하는 차트와 리스트는 아래와 같다.

  • - CPU사용률 : 분석일에 대한 전체 CPU 사용률
  • - Elapsed Time Diff : 정규 SQL과 Unknown SQL 의 수행시간 비교
  • - CPU Time Diff : 정규 SQL과 Unknown SQL 의 CPU시간 비교
  • - Physical Read Diff : 정규 SQL과 Unknown SQL 의 Physical Read 비교
  • - Logical Read Diff : 정규 SQL과 Unknown SQL 의 Logical Read 비교
  • - Sort Diff : 정규 SQL과 Unknown SQL 의 Sort 량 비교
  • - Adhoc TOP SQL : Adhoc SQL 중 activity >= 50 이내에 포함되는 SQL 목록
  • - Adhoc TOP Module : Adhoc SQL을 수행한 모듈 별 성능 통계

정규 SQL 필터링 조건 관리

정규 SQL 필터링 조건을 추가, 삭제 등을 관리한다.

DB를 선택하고 검색을 하면 Parsing Schema Name과 필터 유형 정보를 조회한다.
Parsing Schema Name을 선택하여 추가한다.
필터 유형을 선택하고 필터 조건을 수동 입력하여 추가한다.
Parsing Schema Name과 필터 유형에서 Delete 컬럼으로 조건을 삭제한다.
