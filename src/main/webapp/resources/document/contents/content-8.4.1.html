8. 성능 점검

8.4. SQL 성능 자동 점검

8.4.1. 성능 자동 점검

SQL 자동 성능 점검 대상(특정 OWNER, 특정 테이블, 특정 모듈)에 대해 개발계에서 테스트한 SQL을 테스트계에서 자동 수행하여, 운영에 반영했을 때 성능에 문제가 없는지 자동으로 점검하는 기능으로 프로젝트별 수행회차에 따른 성능 점검 이전 / 이후에 대한 성능 정보를 보여준다. 프로젝트가 선택이 되면 최상위 수행회차가 선택이 되며, 그 회차의 수행결과와 SQL자동성능점검이 진행된 결과를 보여준다.

엑셀

현재 데이터를 엑셀형식으로 다운로드 하여 보여준다.

수행결과

수행회차에 대한 진행 정보를 보여준다.

참고 사항

버퍼 임팩트(%), 수행시간 임팩트(%) 컬럼의 ‘-(마이너스)‘ 기호가 있는 값에 대해서는 성능 향상이 있음을 보여준다.

SQL자동성능점검 확인

SQL자동성능점검을 클릭하면 해당 기능을 진행할 것인지 확인한다.

SQL자동성능점검

SQL자동성능점검은 프로젝트의 SQL자동성능점검 조건에 따라서 수행회차가 생성되고, 해당 정보를 바탕으로 SQL자동성능점검이 진행된다.

SQL자동성능점검 진행중

프로젝트에 대하여 SQL자동성능점검이 진행 중일 때 다시 SQL자동성능점검을 클릭하면 다음과 같은 메시지로 SQL자동성능점검이 진행 중임을 보여준다.

강제완료처리

진행중인 SQL 자동성능점검을 중지해야 할 사유가 발생하면 [강제완료처리] 버튼을 클릭한다. SQL 자동성능점검 조건인 OWNER로 SQL자동성능점검을 모두 강제 완료처리되는 기능을 진행할 것인지 확인하는 메시지 창이 뜨고,

‘YES’를 클릭하면 진행중인 SQL자동성능점검에 대한 모든 데이터를 강제적으로 완료처리한다.

강제완료처리 후 [검색] 버튼을 클릭하면 해당 수행회차에 대한 수행결과에 “강제완료: Y” 상태가 표시된다.

SQL 적재

SQL 자동 성능 점검의 데이터에서 마우스를 클릭하면 SQL 적재 팝업 화면을 보여준다.

권한 스트립트

SQL 성능자동점검에 필요한 Database 권한에 대한 스크립트 작성을 도와주는 팝업 화면을 보여준다.
