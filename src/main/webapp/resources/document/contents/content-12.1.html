12. DB 진단/점검

12.1. 일 예방 점검 – DB 상태 점검

DB 상태 점검은 데이터베이스의 다양한 구성요소에 대한 상태를 점검한다. 점검 항목은 DB상태, 인스턴스 상태, 리스너 상태, 공간 사용률, 오브젝트 상태 등이며 점검에 대한 결과와 함께 문제 해결을 위한 조치 스크립트를 함께 생성한다. 일 예방 점검은 두 가지 방법으로 실행 할 수 있다.

  • - 자동 일 예방 점검 : Open-POP은 매일 정해진 시간에 정기적으로 DB의 상태를 자동으로 점검
  • - 수동 일 예방 점검 : DBA가 수동으로 실행하여 DB상태를 점검

DB 상태 점검 결과는 Open-POP 저장소에 저장되어, 주간 또는 월간 분석을 통해 자주 발생하는 이슈에 대한 특별한 예방 조치를 수행 할 수 있게 도움을 준다.

DB 상태 점검 조회

점검일을 선택하여 일 예방점검 현황을 조회한다.

일 예방 점검은 기본으로 필수 항목에 대한 점검 결과를 화면에 보여주고, 권고성 점검 항목은 토글 버튼을 통해 화면에 보여준다.

“HEALTH” 항목의 의미는 아래와 같다.

  • - Health Green : 조치 또는 확인 대상 없음
  • - Health Red : 조치 또는 확인 대상 있음
  • - Health None : 미점검

점검 상세로 이동

일 예방 점검 상태에 이슈가 있는 경우 해당 항목은 “■” 로 표시되며 항목을 클릭하여 상세 내용을 별도의 탭으로 확인한다.

아래 그림은 [일 예방점검 – INVALID OJBECT]의 상세화면이며, 조치 스크립트를 오브젝트 단위 또는 전체 오브젝트를 대상으로 제공한다.

예외등록

확인 결과 조치가 필요하지 않고 다음 점검부터 제외하고자 할 경우 예외 등록 처리한다. 점검항목별 상세 화면에서 예외 등록하고자 하는 항목을 체크 후 [예외등록] 버튼을 클릭한다.

예외 등록 조회 / 예외 삭제

[예외 관리] 버튼을 클릭하면 ‘일 예방 점검 예외 관리’ 화면으로 이동되며 DB, 점검항목을 선택하고 조회 후 그리드에서 삭제 항목을 체크하고 [예외삭제] 버튼을 클릭한다.
