8. 성능 점검

8.3. SQL 성능 점검

8.3.1. 자동 선정

[자동 선정] 탭

DB단위로 자동선정주기, 선정범위, 임계값 등의 조건을 설정하여 TOP SQL을 선정하고 성능 개선 관리대상으로 자동 등록되도록 스케줄링 한다.

자동선정조건 스케줄 등록

DB 별로 자동선정조건을 조회하여 기 등록된 선정 조건을 변경 또는 삭제하거나 새로 자동선정조건을 등록한다.

통상 온라인 프로그램과 배치 프로그램의 TOP SQL 선정조건이 다르기 때문에 하나의 DB에 여러 개의 자동선정 조건을 등록하여 사용할 수 있다.

설정 항목의 의미는 아래와 같다.

  • - DB : [필수] 선정 대상 데이터베이스, 업무식별자로 사용되어 한번 등록되면 수정할 수 없다.
  • - 튜닝담당자 자동 할당 : [필수] 자동 선정 시 튜닝담당자를 자동 할당 할지 여부. No 인 경우 튜닝담당자(튜너) 지정
  • - 프로그램 유형 : [필수] 온라인/배치 유형 선택
  • - 수집주기 : [필수] 일/주/월 등의 스케줄러 수행 주기
  • - 수집범위 : [필수] 일/주/월/분기/년 등의 TOP SQL의 수행 범위
  • - 선정시작일, 선정종료일 : [필수] 스케줄러 수행 시작과 종료일
  • - 이전 선정 SQL제외 : [필수] 이전에 선정된 SQL을 제외할 지 여부
  • - 이전 튜닝 SQL 제외 : [필수] 이전에 튜닝한 SQL을 제외할 지 여부
  • - Elapsed Time, Buffer Gets, Execution : [필수] TOP SQL 임계값
  • - Module, SQL Text : [선택] TOP SQL 조회시 필터링 조건
  • - Parsing Schema Name : [선택] SQL 수행 파싱스키마명
  • - TOP N : [필수] TOP SQL 최대 건수
  • - Ordered : [필수] 정렬을 위한 기준 항목
  • - 애플리케이션 필터 여부 : [선택]
  • - 사용여부 : [필수] 자동선정 조건의 사용여부
  • - 삭제여부 : [필수] 데이터의 삭제여부

[초기화] 버튼은 자동선정조건 신규 등록을 위하여 입력된 항목을 모두 Clear시킨다.

자동선정조건 일괄 수정

여러 건의 선정조건번호를 일괄 수정할 경우 사용한다.
수정할 선정조건번호를 체크하여 선택한다. [일괄 수정] 탭에서 일괄 수정 대상 항목을 체크하고 수정 값을 입력 후 [일괄 수정] 버튼을 클릭한다. 체크된 일괄 수정 항목에 대해서만 일괄 변경한다.

[자동 선정 현황(선정회차)] 탭

자동선정 조건에 의해 튜닝대상으로 선정되어 튜닝이 진행되고 있는 TOP SQL 목록을 확인한다. 튜닝대상 선정이 잘못되었거나 튜닝이 불필요한 경우 목록을 선택하여 선정회차별 일괄 튜닝종료가 가능하다.

상단 그리드를 클릭하면 선택된 로우의 상세 데이터가 하단 그리드에 조회된다.

[자동 선정 현황(검색)] 탭

자동선정 조건에 의해 튜닝대상으로 선정되어 튜닝이 진행되고 있는 TOP SQL 목록을 확인한다. 튜닝대상 선정이 잘못되었거나 튜닝이 불필요한 경우 선정조건번호 단위로 일괄 튜닝종료가 가능하다.

[자동 선정 이력] 탭

선정조건 변경에 대한 이력 정보를 조회한다.[자동 선정] 탭에서 조회하고자 하는 선정조건번호를 선택하고 [선정조건 변경이력] 버튼을 클릭한다.
