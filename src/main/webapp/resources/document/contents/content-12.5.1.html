12. DB 진단/점검

12.5. SPACE 분석

12.5.1. Tablespace 분석

테이블스페이스의 공간 활용성을 분석한다. DB Size 증가속도와 TOP 테이블스페이스 증가속도를 비교 분석하여 DB Size 증가의 원인이 되는 테이블스페이스와 세그먼트를 찾고 Fragmentation이 크게 발생한 세그먼트를 대상으로 Reorg 작업을 수행할 수 있는 스크립트를 제공한다.

테이블스페이스 분석

테이블스페이스 분석대상 DB를 선택하여 테이블스페이스를 분석하면 세 개 카테고리의 화면이 보여진다.

  • - DB Size Growth : DB 사이즈 증가 추이
  • - TOP Tablespace Growth : TOP 테이블스페이스의 증가 추이를 보여주며 테이블스페이스를 선택하면 상세 분석화면으로 이동
  • - Tablespace 목록 : 전체 Tablespace 목록을 보여주며 테이블스페이스를 선택하면 상세 분석화면으로 이동

테이블스페이스 상세 분석

테이블스페이스 분석화면에서 특정 테이블스페이스를 선택하여 테이블스페이스 상세분석화면으로 이동하며 아래의 네 개 카테고리의 화면이 보여진다.

  • - Segment List : 해당 테이블스페이스에 생성된 전체 세그먼트 목록을 보여주며 Reorg대상인 경우 블루 계열의 백그라운드 색으로 표현
  • - Segment Statistics/Size History : Reorg대상인 경우 Reorg 스크립트와 함께 Physical Read/Write 발생 추이와 사이즈 증가 추이를 보여줌
  • - Datafile List : 해당 테이블스페이스의 데이터파일 목록
  • - Datafile Statistics : 데이터파일의 Physical Read/Write 발생 추이
