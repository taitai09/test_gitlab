9.5. 성능 영향도 진단

9.5.2. 오브젝트변경 SQL성능 영향도 진단

테이블 컬럼 추가/삭제, 인덱스 추가/삭제, 데이터 대량 Load와 같은 시나리오에서 SQL이 성능에 미치는 영향을 진단하는 기능이다.

영향도 진단 현황 조회

DB를 선택한 후 SQL 영향도 진단 현황을 조회하여 DB별 전체 진단 현황을 확인한다.
진단번호를 선택하면 상세화면으로 이동한다.

테이블 별 성능 영향도 현황 조회

SQL 영향도 진단 상세 조회

선택한 테이블의 진단 대상SQL목록과 오브젝트 변경 전/후의 성능 비교표를 통해 성능이 악화된 SQL목록을 확인하고 SQL Profile 적용을 통해 성능이 좋았던 이전 실행계획으로 변경할 수 있다.

성능 임팩트 유형은 3가지로 분류한다.

  • - Unchanged : DB변경전과 성능이 동일
  • - Improved : DB변경전보다 성능이 좋음
  • - Regressed : DB변경전보다 성능이 나쁨

영향도 진단 스케줄 등록

SQL영향도 진단은 오랜 시간이 소요되는 작업이므로 Open-POP 스케줄러에 등록하여 실행하고 실행결과는 Open-POP 화면을 통해 확인한다.

스케줄 등록 항목은 아래와 같다.

  • - DB : 진단 대상 DB
  • - 즉시실행 여부 : 스케줄러에서 등록 즉시 실행시킬 것인지 여부로 즉시실행이 선택되면 작업수행일자가 Disable 됨
  • - SNAP_ID : 진단 대상 SQL의 수행 기간 범위(AWR Snapshot)
  • - 작업수행일자 : 스케줄러 작업 수행일
  • - 작업수행시간 : 스케줄러 작업 수행시간
  • - 테이블 목록 : 영향도 진단 대상 테이블 선택
