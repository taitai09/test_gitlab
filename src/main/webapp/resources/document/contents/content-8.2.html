8. 성능 점검

8.2. SQL 셀프 성능 점검

개발자가 작성한 SQL의 성능 품질을 일정 수준 확보하기 위하여 개발자 스스로 튜닝할 수 있는 환경을 제공한다. 튜닝 체크리스트, 인덱스 설계가이드, 셀프 테스트, SQL 상세성능정보, Explain Plan, Execution Plan, 인덱스 자동설계기능을 제공하며 개발자 스스로 튜닝이 어려운 경우 튜닝 중인 SQL을 바로 튜닝 요청할 수 있도록 성능개선요청 화면으로 연동된다.

이 화면에서 SQL 셀프튜닝과 관련된 모든 기능을 활용할 수 있다.

기능 설명
인덱스 가이드 인덱스 생성/변경/삭제 가이드 및 체크리스트
튜닝체크 리스트 실행계획, SQL구조, 조인방법, 인덱스효율성 측면의 튜닝 체크리스트
Explain Plan Explain Plan 및 상세 테이블정보(테이블정보, 컬럼정보, 인덱스정보, 통계변경 히스토리, Access Path, 테이블 변경 히스토리)
성능 점검 수행 SQL을 실행하고 SQL의 성능 점검 결과를 보여줌
  • 상세 점검 결과 : 적합 또는 부적합 결과
  • 개선가이드 : 점검 지표에 대한 설명 및 개선가이드
  • Execution Plan : 실제 실행계획, Query Block, Outline, Predicate
  • SQL Statistics : Elapsed Time, CPU Time, Buffer Gets, Disk Reads
인덱스 자동설계 Open-POP Index Advisor 가 필요한 인덱스 추천
튜닝요청 개발자 스스로 튜닝이 어려운 경우 튜닝요청 페이지로 이동
SQL Tuning Advisor 오라클의 SQL Tuning Advisor 실행 화면으로 이동
SQL Access Advisor 오라클의 SQL Access Advisor 실행 화면으로 이동

SQL 셀프 테스트 실행

업무, DB, 파싱 스키마를 선택하고 SQL_TEXT와 바인드 값을 입력한 후 [성능 점검 수행] 버튼을 클릭하면 성능 점검 결과, 예상실행계획 정보를 확인할 수 있다.

성능 점검 > 결과

성능 점검 수행 결과 점검 지표의 적합, 부적합 결과를 확인할 수 있다.

성능 점검 > Execution Plan

성능 점검 수행 결과 실제 실행계획을 확인할 수 있다. 블럭수(%)가 높은 행이 튜닝 대상이다.

성능 점검 > Display Cursor

성능 점검 수행 결과 실제 실행계획과 Query Block, Outline, Predicate 정보를 확인할 수 있다.

Explain Plan

Table Info

테이블정보, 컬럼정보, 인덱스정보, 통계정보 변경이력, Access Path, 테이블 변경이력 정보를 확인할 수 있다.

인덱스 자동설계

[인덱스 자동설계] 버튼을 클릭하면 Open-POP Index Advisor 가 인덱스를 추천한다.

  • Data Sampling 기반: 데이터 샘플링 기반 selectivity 계산
  • Statistic 기반: 통계 정보 기반 selectivity 계산

튜닝요청

[튜닝요청] 버튼을 클릭하면 튜닝요청 화면으로 이동하고, 셀프튜닝 시 입력한 정보(SQL Text, SQL BIND)를 튜닝요청 화면에 입력해준다.

인덱스가이드

인덱스 생성/변경/삭제 가이드 및 체크 리스트를 보여준다.

튜닝체크리스트

실행계획, SQL구조, 조인 방법, 인덱스효율성 측면의 튜닝 체크리스트를 보여준다.

SQL Tuning Advisor(DBA 권한의 경우)

SQL Tuning Advisor 화면으로 이동하여 Advisor를 실행할 수 있다.

SQL Access Advisor(DBA 권한의 경우)

SQL Access Advisor 화면으로 이동하여 Advisor를 실행할 수 있다.
