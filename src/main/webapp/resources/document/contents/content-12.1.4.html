12. DB 진단/점검

12.1. 일 예방 점검 – DB 상태 점검

12.1.4. Object 점검

Invalid Object

Invalid Object를 점검한 결과이며 목록과 함께 조치 스크립트를 함께 제공한다. Valid 상태로 변경되지 않는 경우 오브젝트 참조 관계를 검토한 후 재 컴파일하여야 한다.

Nologging Object

Nologging Object를 점검한 결과이며 목록과 함께 조치 스크립트를 함께 제공한다. Archive Log 을 생성하지 않기 때문에 복구 누락이 될 수 있으므로 Logging 상태로 변경해야 한다.

Parallel Object

Parallel Object를 점검한 결과이며 목록과 함께 조치 스크립트를 함께 제공한다. 오브젝트 단위의 Parallel 옵션은 관련 SQL에 모두 적용되기 때문에 매우 위험하므로 Noparallel 로 변경한다.

Unusable Index

Unusable Index를 점검한 결과이며 목록과 함께 조치 스크립트를 함께 제공한다. Index를 사용하는 SQL의 성능저하가 우려되므로 Rebuild를 통해 Usable상태로 변경한다.

Chained Rows

Chain이 발생한 건수가 전체 건수의 1% 이상인 테이블을 점검한 결과이다. Chain이 많이 발생한 경우 I/O 가 많아지므로 Reorg작업을 수행하는 것을 권고한다.

Corrupted Block

손상된 블록을 점검한 결과이다. 손상된 블록은 dbms_repair 프로시저를 사용하여 데이터 블록을 손상된 데이터 블록으로 표시하여 해당 테이블을 사용하는 쿼리를 유지할 수 있다. 계속해서 손상된 블록으로 보고될 경우 아래와 같이 rebuild를 권고한다.

Sequence

설정한 임계치에 근접한 시퀀스 목록이다. 시퀀스 max값 변경이 필요하다.

Foreign Key Without Index

Foreign Key에 인덱스가 없는 오브젝트를 점검한 결과이다. Foreign Key에 인덱스가 없으면 DML작업시 TM락(테이블락) 발생으로 동시성이 저하된다.

Disabled Constraint

Disable된 제약조건 목록이다. 제약조건이 disable되어 있는 경우 데이터 정합성이 훼손될 수 있으므로 enable상태로 변경해야 한다.
