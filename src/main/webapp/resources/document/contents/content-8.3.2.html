8.3. SQL 성능 점검

8.3.2. 수동 선정

[수동 선정] 탭

DB, 임계값 등의 조건을 설정하여 TOP SQL을 조회한 후 튜닝요청을 통해 성능개선관리 대상으로 등록한다.

설정 항목의 의미는 아래와 같다.

  • - DB : [필수] 선정 대상 데이터베이스, 업무식별자로 사용되어 한번 등록되면 수정할 수 없다.
  • - SNAP ID : [필수] 선정 대상 수집 범위를 지정하는 시작과 종료 SNAP ID
  • - 수동선정이력 : 해당 DB 의 수동선정 이력을 팝업창으로 보여준다.
  • - 이전 튜닝대상 선정 SQL제외 : [필수] 이전에 선정된 SQL을 제외할 지 여부
  • - Elapsed Time, Buffer Gets, Executions : [필수] TOP SQL 임계값
  • - TOP N : [필수] TOP SQL 최대 건수
  • - Ordered : [필수] 정렬을 위한 기준 항목
  • - Module, Parsing Schema Name, SQL TEXT : [선택] TOP SQL 조회시 필터링 조건
  • - Module(일괄제외), Parsing Schema Name(일괄제외), SQL_ID(일괄제외) : [선택] TOP SQL 조회시 필터링 제외 조건

Filter SQL

사용자가 직접 Module, Parsing Schema Name, SQL ID, Plan Hash Value 항목에 대한 SQL을 추가할 수 있다.

튜닝 대상 선정

프로젝트, 튜닝진행단계는 선택 항목이다.

[수동 선정 이력] 탭

수동선정 조건에 의해 튜닝대상으로 선정되어 튜닝이 진행되고 있는 TOP SQL 목록을 확인한다. 튜닝대상 선정이 잘못되었거나 튜닝이 불필요한 경우 목록을 선택하여 일괄 튜닝종료가 가능하다.

상단 그리드를 클릭하면 선택된 로우의 상세 데이터가 하단 그리드에 조회된다.
