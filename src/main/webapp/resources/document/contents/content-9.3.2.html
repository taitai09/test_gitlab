9.3. SQL 성능 분석

9.3.2. SQL 성능 분석

SQL_ID, PLAN_HASH_VALUE조건으로 SQL의 상세 성능을 분석한다. 데이터 증가에 따른 수행시간 증가, 입력되는 바인드 값에 따른 성능차이, 플랜변경에 의한 성능차이를 SQL성능분석을 통해 쉽게 분석할 수 있다.

SQL 성능 조회

DB, SQL_ID, PLAN_HASH_VALUE(옵션)를 입력한 후 검색하면 아래와 같이 조회된다.

  • - Bind Value : 쿼리 실행에 사용된 바인드 값과 바인드 값 별 성능정보
  • - Outline : 쿼리 실행 시에 내부적으로 사용된 Outline정보(Hint)
  • - 유사 SQL : SQL Text 가 유사한 SQL목록과 각각의 성능정보
  • - Tree Plan, Text Plan, Grid Plan : 다양한 포맷의 Plan
  • - AWR SQL Stats : AWR의 SQL 성능 이력
  • - Plan별 SQL Stats : Plan별 SQL의 성능정보
  • - AWR Execution Plan : AWR Execution Plan 정보

튜닝요청

[PLAN별 SQL Stats] 탭에서 SQL을 선택한 후에 튜닝 요청한다.

SQL Profile 적용

PLAN 이 여러 개 인 경우 SQL Profile 적용을 통해 Good Plan으로 플랜을 고정한다.

SQL Tuning Advisor

SQL Tuning Advisor 를 통해 오라클의 튜닝 권고 내용을 확인할 수 있다.

SQL Access Advisor

SQL Access Advisor를 통해 오라클의 인덱스 권고 내용을 확인할 수 있다.

SQL Monitor

SQL 이 현재 수행 중인 경우 SQL Monitor를 통해 상세분석 할 수 있다.
