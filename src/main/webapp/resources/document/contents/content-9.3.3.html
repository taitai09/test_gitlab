9. 성능 최적화

9.3. SQL 성능 분석

9.3.3. 테이블/인덱스 사용 SQL 분석

[테이블 사용 SQL 분석] 탭

LOCK, I/O 경합 등의 성능적인 이슈가 발생하여 특정 테이블을 참조하는 SQL에 대한 상세 분석이 필요한 경우, 테이블 사용 SQL분석을 통해 성능 이슈의 원인을 발견하고 이슈를 해결하는데 유용하게 활용할 수 있다.

SQL현황조회

DB, OWNER, TABLE_NAME 을 입력하여 SQL목록을 조회한다.

SQL 성능분석

SQL FULL Text, Bind값에 따른 성능정보, Plan, Outline, SQL Statistics History 등의 상세성능정보를 확인한다.

[인덱스 사용 SQL 분석] 탭

인덱스 경합으로 인한 성능 저하가 발견되면 해당 인덱스를 참조하는 모든 SQL목록을 확인하여 이슈의 원인을 발견하고 이슈를 해결하는데 유용하며, 변경/삭제와 같은 인덱스 정비 작업 시 참조하는 인덱스 현황을 확인할 수 있어 작업 영향도를 최소화할 수 있다.

SQL현황조회

DB, OWNER, INDEX_NAME 을 입력하여 SQL목록을 조회한다.

SQL 성능분석

SQL FULL Text, Bind값에 따른 성능정보, Plan, Outline, SQL Statistics History 등의 상세성능정보를 확인한다.
