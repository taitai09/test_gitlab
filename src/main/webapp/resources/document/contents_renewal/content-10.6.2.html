10. 종합 진단

10.6. 신규 SQL 성능 변화 분석

10.6.2. 자동 검증 배포 SQL

개요

배포전 자동 성능 검증을 통해 운영에 적용된 신규 또는 변경 SQL의 성능을 운영 환경에서 추적 관리하여, 지속적인 성능저하 현상이 감지되는 지 검증한다.

화면 구성

화면은 다음과 같은 탭으로 구성되어 있다.

Tab 명 Tab 설명
SQL 성능 추적 현황
성능 검증 SQL
  • 성능 검증을 수행 후 적합 판정을 받고 운영에 적용된 SQL의 성능 검증 현황을 제공한다.
예외 처리 SQL
  • 영구 / 한시 검증제외 예외 처리를 하고 운영에 적용된 SQL의 성능 검증 현황을 제공한다.
Charts
  • Buffer Get, Elapsed Time, 성능 검증 SQL, 예외 처리 SQL의 관점에서 성능 검증 현황에 대한 정보를 Chart로 제공한다.
SQLs
  • 개별 SQL에 대한 성능 추적, Plan 정보, 성능 점검 결과 비교, 프로그램 정보, 성능 추이 정보를 제공한다.
[SQL 성능 추적 현황] 탭 > [성능 검증 SQL] 탭

성능 자동 검증 수행 후 적합 판정을 받고 운영에 적용된 SQL의 성능을 검증한다. 배포일로 부터 현재 시점까지 추적하여 배포전 대비 성능 향상, 성능 저하, 부적합에 대한 현황 정보를 조회한다.

튜닝대상 선정 전체 선택

검색된 전체 SQL(모든 페이지)에 대해 튜닝대상으로 선정한다. 선정된 튜닝대상에 대한 튜닝 작업은 ‘성능 최적화’ 메뉴에서 진행한다.

  1. ‘전체’ 체크 박스를 클릭하면 하위의 모든 체크 박스가 체크된다.
