13. DB 운영관리

13.1. 장애 예측(자원한계점) 분석

CPU 사용량 이상 증가 원인 예측, CPU / Sequence / Tablespace 사용 한계점 도달 시점 예측, 신규 애플리케이션/신규 SQL 타임아웃 시점 예측, Unknown(Adhoc) SQL 장애 원인 예측을 통해 미리 사전에 장애가 발생될 시점과 원인 정보를 제공한다.

13.1.1. CPU 사용량 증가원인 예측

데이터베이스의 정상적인(최적) 운영 상태인 baseline 설정 정보 기준으로 기준일(조회일)에 CPU 사용량이 증가한 상세 원인 정보를 제공한다. 기준일 선택은 평상시 대비 CPU사용량이 과도하게 증가한 일자를 선택하며, 분석의 범위를 좁히기 위해 시간 구분을 사용한다.

CPU 사용량 증가 원인 예측

예측 기준 / 기준일 / 시간 구분 등의 조건을 선택하여 검색하면 CPU 사용량이 예측 기준보다 증가한 인스턴스 목록과 CPU 사용량(%)을 그래프로 제공한다.

CPU사용량 증가원인 예측 차트 범례의 의미는 아래와 같다.

  • - 베이스라인 CPU 사용률 : 화면에서 선택한 기준일 이전의 최근 baseline 설정 정보 (베이스라인 BETWEEN 기준일 – baseline 설정 정보 AND 기준일 - 1)
  • - 기준일 CPU 사용률 : CPU 증가원인을 분석하고 문제 요인을 찾기 위하여 화면에서 선택한 기준일

CPU 사용량 증가 원인 예측 상세

앞서 제공된 그래프를 선택하면 상세하게 조회된 정보를 그래프로 제공된다.
상세 화면에서 Time Model 바차트 그래프를 선택하면 선택된 Time Model 증가의 원인에 대한 TOP SQL 정보를 보여준다.

차트 및 리스트 항목은 아래과 같다.

  • - CPU 사용률 : 베이스라인 CPU사용률과 분석일 CPU 사용률 비교
  • - User CPU 사용률 : 베이스라인 User CPU사용률과 분석일 User CPU 사용률 비교
  • - SYS CPU 사용률 : 베이스라인 SYS CPU사용률과 분석일 SYS CPU 사용률 비교
  • - Time Model : 베이스라인 Time Model 과 분석일 Time Model 비교
  • - TOP SQL : Time Model 과 관련된 Activity 가 50% 이상인 SQL 목록
