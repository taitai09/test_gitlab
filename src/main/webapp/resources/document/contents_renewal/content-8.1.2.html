8. 품질 점검

8.1. SQL 표준 일괄 점검

8.1.2. 실행기반 SQL 표준 점검 작업

개요

테스트 서버에서 실행된 SQL에 대한 표준을 일괄 점검한다. 등록된 스케줄에 의해 SQL이 수집되고 점검을 진행한다. 수행된 표준 점검 결과에 대한 집계 정보 및 표준 미준수 SQL의 상세 정보를 제공한다.

화면 구성

화면은 다음과 같은 탭으로 구성되어 있다.

Tab 명 Tab 설명
스케줄러 관리
  • 점검을 진행할 SQL 대상을 설정하고, SQL 표준 일괄 점검을 수행할 스케줄을 등록/수정/삭제한다.
SQL 표준 점검 결과
  • 수집된 SQL에 대한 표준 일괄 점검 진행 상황 및 결과를 조회한다. 점검 수행은 등록된 스케줄에 의해 자동 실행된다.
  • SQL 표준 점검 수행 결과에 대한 집계 정보 및 미준수 SQL의 상세한 정보를 엑셀로 다운로드 받을 수 있다.
표준 미준수 SQL
  • 점검 수행 결과 표준을 미준수한 SQL 정보 및 미준수 품질지표 정보를 조회할 수 있다.
SQL 표준 준수 현황
  • 점검 일자별 표준 미준수 본수 및 표준 미준수율, 품질지표별 미준수 본수에 대한 현황을 그래프로 보여준다.
SQL 표준 점검 예외 대상 관리
  • 표준 점검 결과 예외로 처리되어야 할 대상을 관리(조회/등록/수정/삭제)한다.

SQL 표준 점검을 수행하기 전에 기준 정보가 등록되어야 한다. 기준 정보 등록은 신규 프로젝트 등록 > 프로젝트별 SQL 품질지표 선정 > 프로젝트별 SQL 품질점검 RULE 등록 순으로 진행한다.

  1. ① 신규 프로젝트 등록: 설정 > 기준정보 설정 > 프로젝트 관리 > [프로젝트 관리] 탭 메뉴 이용(이미 프로젝트가 등록되었을 경우 생략)
  2. ② 프로젝트별 SQL 품질지표 선정: 품질 점검 > SQL 표준 점검 > SQL 표준 점검 설정 > 프로젝트 SQL 품질점검 지표 관리] 탭 메뉴 이용
  3. ③ 프로젝트별 SQL 품질점검 RULE 등록: 품질 점검 > SQL 표준 점검 > SQL 표준 점검 설정 > [프로젝트 SQL 품질점검 RULE 관리] 탭 메뉴 이용

수집기간
  • SQL을 수집할 기간
    • 주기
      일: D-1 수집
      주: D-1 ~ 이전 1주일
      월: D-1 ~ 이전 1 개월
    • 기간: 사용자가 직접 수집 시작일자와 수집 종료일자를 입력
Table Owner in
  • 특정 OWNER에 대한 수집이 필요한 경우 입력한다.
  • N개의 OWNER를 입력할 경우 콤마(,)로 분리하며 대소문자는 구분하지 않으며 최대 100개까지 입력 가능하다.
    예) ORACLE, OpenPOP, orange
  • 입력된 OWNER와 동일한 SQL에 대해서만 수집한다. 입력된 OWNER는 내부적으로 아래와 같은 조건으로 필터링에 포함된다.
  • WHERE UPEER(OWNER) IN (‘ORACLE’, ‘OPENPOP’, ORANGE’)
Module like
  • 소스 DB에서 특정 MODULE에 대한 수집이 필요한 경우 입력한다.
  • N개의 MODULE을 입력할 경우 콤마(,)로 분리하며 대소문자는 구분하지 않으며 최대 100개까지 입력 가능하다.
    예) JDBC, dbsnmp
  • 입력된 MODULE이 포함된 SQL에 대해서만 수집한다. 입력된 MODULE은 내부적으로 아래와 같은 조건으로 필터링에 포함된다.
  • WHERE (UPEER(MODULE) LIKE ‘%JDBC%’ OR UPEER(MODULE) LIKE ‘%DNSMP%’)
Filter SQL

Filter SQL

  • [Filter SQL] 버튼을 클릭한다.
  • Filter SQL 팝업 창이 뜨고, Module, Action, Parsing Schema Name, SQL_TEXT, SQL ID 항목에 대한 SQL을 추가한 후 [저장]버튼을 클릭한다.
  • 저장된 내용이 [Filter SQL] 버튼 옆에 보여진다.
  • SQL_TEXT 선택 시 LIKE , NOT LIKE 만 선택 가능하다.

수행 결과 집계 및 미준수 SQL 엑셀 다운로드

  1. 다운로드 받을 스케줄러를 클릭한다.
  2. 하단-좌측 그리드 행을 클릭하면 하단-우측 그리드에 수행 결과 집계가 조회된다.
  3. [엑셀] 버튼을 클릭한다.

NOTE

다운로드 된 엑셀 파일은 3개의 Sheet로 구성되어 있다.

  • SQL 품질검토 작업 집계표 Sheet: 표준 점검 결과 집계 현황
  • 표준 미준수 SQL Sheet: 표준을 미준수 한 SQL. 미준수한 점검지표가 표시됨
  • 표준 미준수 개발자 집계표: 개발자 별 표준을 미준수 한 SQL 수 집계 현황

등록

  1. 초기화 버튼을 클릭한다.
  2. 관련 정보를 입력하고 [저장] 버튼을 클릭한다.

NOTE

[품질 점검 지표 코드]는 필수로 입력되어야 하며 [업무 코드], [SQL 식별자(DBIO)], [디렉토리명] 중 하나를 입력하거나, [표준점검DB] 와 [SQL ID]가 입력되어야 한다.

수정

  1. 수정하고자 하는 행을 클릭한다.
  2. 선택된 값이 조회되며, 변경할 값을 입력하고 [저장] 버튼을 클릭한다.

삭제

  1. 삭제하고자 하는 행을 클릭한다.
  2. 선택된 값이 조회되며, [삭제] 버튼을 클릭한다.
일괄 업로드

대량의 데이터를 예외 처리할 경우 엑셀을 이용하여 일괄 업로드 한다. 업로드 후 성공 건수, 실패 건수, 실패 행에 대해 보여준다.

일괄 업로드

  1. 데이터를 업로드 할 수 있는 양식을 다운받는다.
  2. 다운 받은 템플릿 파일에 예외처리 할 데이터를 입력 후 저장한다.

    NOTE

    [품질 점검 지표 코드]는 필수로 입력되어야 하며 [업무 코드], [SQL 식별자(DBIO)], [디렉토리명] 중 하나를 입력하거나, [표준점검DB] 와 [SQL ID]가 입력되어야 한다.

  3. [Choose File] 버튼 클릭, 예외 처리할 엑셀 파일을 선택 후 [엑셀 업로드] 버튼을 클릭한다.
