14. 설정

14.3. 기준정보 설정

14.3.8. 프로젝트 관리

프로젝트, 프로젝트 대상 업무, DB, 튜닝진행단계 정보를 관리한다.

[프로젝트 관리] 탭

프로젝트를 조회하고 신규로 추가하거나 수정할 수 있다.

프로젝트 조회

화면의 상단에서 조회 조건으로 목록을 조회할 수 있다.

프로젝트 신규 추가

화면 하단에서 [초기화] 버튼을 클릭하고 신규로 추가할 프로젝트 정보를 입력한 후 [저장] 버튼을 누른다.

프로젝트 수정

화면의 상단에 있는 목록에서 수정할 프로젝트를 선택하면 하단에 선택한 정보가 나타나고 수정할 내용을 변경한 후 [저장] 버튼을 누른다.

프로젝트 종료

화면의 상단에 있는 목록에서 삭제할 프로젝트를 선택하면 하단에 선택한 정보가 나타나고 ‘종료여부’를 ‘Y’로 선택후 [저장] 버튼을 누른다.

[프로젝트 업무 관리] 탭

프로젝트 대상 범위의 업무를 조회하고 추가/삭제할 수 있다.
프로젝트 업무 등록 시 업무의 계층구조를 표현하기 위해 상위 업무를 반드시 등록해야 한다.

프로젝트 업무 조회

화면의 상단에서 조회 조건으로 목록을 조회할 수 있다.
돋보기를 클릭하면 프로젝트명으로 조회할 수 있는 팝업창이 뜬다.

프로젝트 업무 신규 추가

화면 하단에서 [초기화] 버튼을 클릭하고 신규로 추가할 프로젝트 업무 정보를 입력한 후 [저장] 버튼을 누른다.

프로젝트 업무 삭제

화면의 상단에 있는 목록에서 삭제할 업무를 선택하면 하단에 선택한 정보가 나타나고 [삭제] 버튼을 누른다.

[프로젝트 DB 관리] 탭

프로젝트 대상 범위의 DB 정보를 조회하고 신규로 추가하거나 수정/삭제할 수 있다.
SQL 성능 자동 점검 기능 이용시 반드시 등록되어야 한다.

프로젝트 DB 조회

화면의 상단에서 조회 조건으로 목록을 조회할 수 있다.
돋보기를 클릭하면 프로젝트명으로 조회할 수 있는 팝업창이 뜬다.

프로젝트 DB 신규 추가

화면 하단에서 [초기화] 버튼을 클릭하고 신규로 추가할 프로젝트 DB 정보를 입력한 후 [저장] 버튼을 누른다. SQL 성능 자동점검 대상 업무의 경우 ‘SQL성능자동점검 대상 여부’ 값을 ‘Y’로 선택한다.

  • - 성능점검원천 DB : 성능점검 대상 SQL을 수집할 DB
  • - 성능점검대상 DB : 성능점검을 수행할 DB

프로젝트 DB 수정

화면의 상단에 있는 목록에서 수정할 업무를 선택하면 하단에 선택한 정보가 나타나고 수정할 내용을 변경한 후 [저장] 버튼을 누른다.

프로젝트 DB 삭제

화면의 상단에 있는 목록에서 삭제할 DB를 선택하면 하단에 선택한 정보가 나타나고 [삭제] 버튼을 누른다.

[프로젝트 튜닝진행단계] 탭

프로젝트 튜닝진행단계별 실적을 조회하기 위해 튜닝진행단계를 신규로 추가하거나 수정/삭제할 수 있다. 튜닝진행단계를 별도로 관리할 필요가 없을 경우 등록할 필요가 없다.

프로젝트 튜닝진행단계 조회

화면의 상단에서 조회 조건으로 목록을 조회할 수 있다.

프로젝트 튜닝진행단계 추가

화면 하단에서 [초기화] 버튼을 클릭하고 신규로 추가할 튜닝진행단계 정보를 입력한 후 [저장] 버튼을 누른다.

프로젝트 튜닝진행단계 수정

화면의 상단에 있는 목록에서 수정할 튜닝진행단계를 선택하면 하단에 선택한 정보가 나타나고 수정할 내용을 변경한 후 [저장] 버튼을 누른다.

프로젝트 튜닝진행단계 삭제

화면의 상단에 있는 목록에서 삭제할 튜닝진행단계를 선택하면 하단에 선택한 정보가 나타나고 ‘삭제여부’를 ‘Y’로 선택후 [저장] 버튼을 누른다.
