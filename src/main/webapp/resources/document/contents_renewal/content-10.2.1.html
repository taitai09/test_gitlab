10.2. 종합 진단

10.2.1 DB/SQL 종합 진단

개요

주간 및 월간 단위로 DB 진단, 오브젝트 진단, 파라미터 진단, SQL 성능 진단, 장애 예측 분석 결과에 대한 진단 결과 요약 및 조치 대상에 대한 상세한 DB 종합 진단 결과를 보고서로 제공한다.

화면 구성

화면은 다음과 같은 탭으로 구성되어 있다.

Tab 명 Tab 설명
진단 결과 요약 • DB 진단, 오브젝트 진단, 파라미터 진단, 장애 예측(자원한계점) 분석 결과에 대한 요약 정보를 제공한다.
DB 진단 • Expired 계정, 파라미터 변경, DB File 생성률 등 DB에 대한 상세 진단 결과를 제공한다.
오브젝트 진단 • Reorg 대상, 파티셔닝 대상, 인덱스 사용 진단 등 오브젝트에 대한 진단 결과를 제공한다.
파라미터 진단 • 공용 파라미터를 제외한 인스턴스 간 불일치 파라미터에 대한 진단 결과를 제공한다.
SQL 성능 진단 • 성능진단 임계값을 기반으로 플랜 변경, 악성 신규 SQL, Temporary 과다 사용 등 DB성능 장애를 유발할 수 있는 다양한 유형의 SQL에 대한 상세 진단 결과를 제공한다.
장애 예측(자원한계점) 분석 • CPU 한계점 예측, Sequence 한계점 예측, Tablespace 한계점 예측, 신규 SQL 타임아웃 예측에 대한 진단 결과를 제공한다.
SQL Text List • SQL_ID와 SQL문 전체를 제공한다.
• SQL 성능 진단, SQL 품질 진단, 신규 SQL 타임아웃 예측 탭에서 진단된 SQL은 SQL_ID만 제공된다. 전체 SQL을 보기 위해서는 해당 SQL_ID를 Ctrl+C하여 현재 탭에서 Ctrl+F를 이용해 찾을 수 있다.
진단 개요 • 진단 일시, 진단 기간, 진단 대상 DB에 대한 정보를 제공한다.

진단 실행 Area

진단 실행

  •   진단을 실행할 DB 선택 > 기준일자 선택 > 주간 진단 또는 월간 진단 선택 후 [진단 실행] 버튼을 클릭한다.

  •   Progressbar가 팝업되고 진단이 완료되면 진단 결과가 조회된다.

보고서 다운로드

  •   진단 실행 후 [다운로드] 버튼을 클릭한다.

  •   압축 파일이 다운로드 되고 해당 파일의 압축을 해제한 후, DB_report 폴더의 db_report.html을 클릭한다.

  •   진단 결과 보고서가 브라우저에 표시된다.

목차 Area

  •   진단 결과가 조회되면 좌측 화면에 전체 목차가 조회된다. 목차를 클릭하면 해당 페이지로 이동한다.

  •    조치 필요, 긴급 조치, 확인 필요 건인 경우 목차 제목 옆에 범례가 표시된다.

  •    [조치 필요] 버튼을 클릭하면 긴급성 조치 필요 건과, 비 긴급성 조치 필요 건에 대해서만 목차에 조회된다

진단 결과 Area

[진단 결과 요약] 탭

  • DB 진단, 오브젝트 진단, 파라미터 진단, SQL 성능 진단, 장애 예측(자원한계점) 분석 결과에 대한 요약 정보를 제공한다.

  •   긴급조치 , 조치필요, 확인필요에 대한 요약 건수가 조회된다.

  •    진단 결과에 대한 범례는 아래와 같다.

    • 우수( ) : 조치 사항 없음
    • 양호( ) : 금일 조치 사항이 없지만 1개월 동안 조치 사항 있음
    • 조치필요( ) : 비 긴급성 조치 필요
    • 긴급조치( ) : 긴급성 조치 필요
    • 확인필요( ) : 진단결과 확인 필요

  • 진단항목을 클릭하면 해당 페이지로 이동하며 상세한 결과가 조회된다

NOTE

■    예외 등록 / 삭제

확인 결과 조치가 필요하지 않고 다음 진단 부터 제외하고자 할 경우 예외 등록을 한다. 예외등록 / 예외삭제는 ‘종합 진단 > 일 예방 진단’ 메뉴에서 처리하며 처리 방법은 해당 매뉴얼을 참조한다



[DB 진단] 탭

Expired 계정, 파라미터 변경, DB File 생성률 등 DB에 대한 상세 진단 결과를 제공한다.

[오브젝트 진단] 탭

Reorg 대상, 파티셔닝 대상, 인덱스 사용 진단 등 오브젝트에 대한 진단 결과를 제공한다.

[파라미터 진단] 탭

  • 공용 파라미터를 제외한 인스턴스 간 불일치 파라미터에 대한 진단 결과를 제공한다.

[SQL 성능 진단] 탭

  • 성능진단 임계값을 기반으로 플랜 변경, 악성 신규 SQL, Temporary 과다 사용 등 DB성능 장애를 유발할 수 있는 다양한 유형의 SQL에 대한 상세 진단 결과를 제공한다.

[장애 예측(자원한계점) 분석] 탭

  • CPU 한계점 예측, Sequence 한계점 예측, Tablespace 한계점 예측에 대한 진단 결과를 제공한다.

[SQL Text List] 탭

  • SQL_ID와 SQL문 전체를 제공한다. SQL 성능 진단, SQL 품질 진단, 신규 SQL 타임아웃 예측 탭에서 진단된 SQL은 SQL_ID만 제공된다. 전체 SQL을 보기 위해서는 해당 SQL_ID를 Ctrl+C하여 현재 탭에서 Ctrl+F를 이용해 찾을 수 있다.

[진단 개요] 탭

  • 진단 일시, 진단 기간, 진단 대상 DB에 대한 정보를 제공한다.
