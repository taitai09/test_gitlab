15.1.6. 공통 화면

Table Info 팝업

화면은 다음과 같은 탭으로 구성되어 있다.

Tab 명 Tab 설명
Table Detail
  • Table 통계, Column, Index 정보를 제공한다.
Statistics History
  • Table 통계 History 정보를 제공한다.
Access Path
  • V$에서 수집된 해당 Table을 사용하는 Access Path 정보를 제공한다.
  • ‘수집SQL 인덱스 설계’ 메뉴를 이용해 수집 시 조회가 가능하다.
Change History
  • Column, Index 생성/변경/삭제에 대한 History 정보를 제공한다.

검증 SQL 목록 팝업

SQL식별자(DBIO)로 조회된 SQL ID별 성능검증 결과를 확인 및 SQL, Bind Value, Execution Plan 확인이 가능하다. 튜닝 필요 시 튜닝요청을 할 수 있다.

화면 구성

SQL ID 목록 Area

SQL식별자(DBIO)로 수행된 SQL ID 목록이 조회된다.

  1. 부적합 체크 시 성능검증결과 ‘부적합’으로 검증된 SQL ID가 조회된다.
    부적합 체크 해제 시 적합, 부적합으로 검증된 전체 SQL ID가 조회된다.
  2. 그리드 클릭 시 해당 SQL ID 에 대한 검증 결과 및 수행된 SQL 정보가 [상세 검증 결과] 탭과 [SQL] 탭에 조회된다.

SQL ID 목록 컬럼

컬럼 컬럼 설명
SQL_ID
  • SQL_ID
수행일시
  • 해당 SQL ID가 최종 수행된 일시
검증결과
  • 검증 결과에 대한 값
    • 적합: 성능 검증 결과 적합
    • 부적합: 성능 검증 결과 부적합
상세 검증 결과 탭

성능 검증 결과 컬럼

컬럼 컬럼 설명
상세 검증 결과
검증지표
  • 성능 검증 지표명
적합
  • 적합 판단을 하는 기준값. 기준값 이하일 경우 적합
성능 검증 결과값 실제
  • 성능 검증 시 실제 결과값
성능 검증 결과
  • 성능점검지표 기반의 성능점검 결과값(적합 또는 부적합)
  • ‘적합’ 값과 ‘실제값’값을 비교하여 적합 또는 부적합을 판단
  • 부적합 건에 대해서는 조치가 필요
예외등록여부
  • 예외로 등록되었는지 여부(Y: 예외 등록)
성능 검증 결과 내용
  • 부적합 시 부적합 원인 및 조치 방법 기술
개선가이드
검증지표
  • 성능 검증 지표명
지표설명
  • 성능 검증 지표에 대한 설명
개선가이드
  • 부적합 지표에 대한 개선 방법을 가이드
Execution Plan
COST(%)
  • 실행 단계별 COST(%)의 비율. 누적값이며 해당 라인에서의 COST(%)를 알고 싶다면 이전 단계의 수행시간(%)값을 빼면된다.
CPU_COST(%)
  • 실행 단계별 CPU_COST(%)의 비율. 누적값이며 해당 라인에서의 CPU_COST(%)를 알고 싶다면 이전 단계의 수행시간(%)값을 빼면된다.
IO_COST(%)
  • 실행 단계별 IO_COST(%)의 비율. 누적값이며 해당 라인에서의 IO_COST(%)를 알고 싶다면 이전 단계의 수행시간(%)값을 빼면된다.
ID
  • 플랜에 순서대로 부여되는 번호
OPERATION
  • 실행 Operation명
COST
  • 해당 Operation에서 CPU_COST 와 IO_COST를 더하여 계산한 예측 비용
CPU_COST
  • 해당 Operation에서 CPU_COST 와 IO_COST를 더하여 계산한 예측 비용
CPU_COST
  • 해당 Operation 작업에 필요한 CPU 사용량을 예측한 비용
  • 다른 Operation 보다 CPU 비용이 높은 경우 부하지점
IO_COST
  • 해당 Operation 작업에 필요한 디스크 I/O 량을 예측한 비용
  • 다른 Operation 보다 IO 비용이 높은 경우 부하지점
SQL 탭

수행된 SQL ID의 SQL문 및 Bind Value를 확인할 수 있으며 튜닝을 요청할 수 있다.

튜닝요청

  1. 튜닝 필요 시 [튜닝요청] 버튼을 클릭한다. 이미 튜닝요청 된 SQL ID는 재요청할 수 없다.
  2. 튜닝요청 건에 대한 진행상황은 메뉴 ‘성능 개선 관리’에서 조회가 가능하다.
