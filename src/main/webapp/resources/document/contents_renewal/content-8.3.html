8. 품질 점검

8.3. SQL 튜닝 사례/가이드

튜닝 과정에서 자주 발생하는 잘못된 SQL 패턴이나 개선 사례를 자동 게시하여 모든 개발자들이 쉽게 확인할 수 있도록 한다. 사례/가이드를 통해 개발자가 작성한 SQL의 품질을 향상시켜 성능과 튜닝 생산성을 높일 수 있다.

우측 상단의 아이콘을 클릭한다.

[SQL 튜닝 가이드] 탭

튜닝 가이드 목록 조회

등록일자, 검색조건을 입력하고 [검색] 버튼을 클릭한다. 중요한 가이드는 뒤로 밀려나지 않고 상위에 게시되도록 상단공지 기능을 제공한다.

목록을 선택하면 가이드 내용을 확인하고 수정 또는 삭제할 수 있는 상세페이지로 이동한다.

튜닝 가이드 수정/삭제

제목, 내용을 수정하거나 첨부파일을 추가/삭제할 수 있으며 상단공지 여부를 선택할 수 있다.

튜닝 가이드 등록

[가이드 등록] 버튼을 클릭하여 웹 에디터를 통해 가이드 내용과 첨부파일을 등록한다.

[SQL 튜닝 사례] 탭

튜닝 사례 목록 조회

등록일자, 검색조건을 입력하고 [검색] 버튼을 클릭한다.

SQL 요청 상세 / SQL 개선 상세

목록을 선택하면 SQL 요청 상세 / SQL 개선 상세 페이지로 이동한다.

튜닝 사례 등록

튜닝 사례 등록은 튜너가 튜닝 완료 시점에 등록한다. 등록 메뉴는 성능 최적화 > 성능 최적화 관리 > 성능 개선 관리 > 성능 개선 관리 상세 > [개선 프로세스] 탭에서 등록한다. (9.1.2 성능 개선 관리 설명 참조)
