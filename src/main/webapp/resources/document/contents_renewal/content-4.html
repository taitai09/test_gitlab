4. 사용자 유형

Open-POP 은 사용자를 7 가지 역할로 분류하며 , 각 사용자 역할에 따라 사용할 수 있는 기능을 정의하였다.

역할 설명
관리자 DB 의 전반적인 현황을 관리하는 역할이다.
  • 일 예방점검현황
  • SQL 진단현황
  • 성능개선진행현황
DBA DB 의 전반적인 성능을 관리하는 역할이다.
  • 일 예방 점검 및 장애 예측분석
  • SQL 성능 진단 검토 및 응급조치
  • SPACE 진단 검토
  • SQL 영향도 진단
  • SQL 성능 분석 및 튜닝 요청
  • SPACE 분석
  • 파라미터 변경분석
  • SQL 튜닝 실적관리
  • 기준정보 설정
튜너 DB 성능 및 SQL 튜닝과 관련된 업무를 처리하는 역할이다.
  • 인덱스 최적화 설계
  • SQL 성능 분석 및 튜닝
  • DB 성능 분석
  • 장애 예측분석
개발자 SQL 튜닝요청 , 튜닝결과적용 , 셀프 튜닝을 통한 애플리케이션의 성능 개선 등의활동을 수행하는 역할이다.
  • SQL 튜닝요청
  • SQL 튜닝결과적용
  • 셀프 튜닝
  • 배포전 성능점검
배포담당자 배포전 성능점검 대상 프로그램에 대한 성능지표관리 , 예외관리등의 관리 업무를처리하는 역할이다.
  • 배포전 성능점검
업무배포
담당자		 해당 업무의 배포전 성능점검 대상 프로그램에 대하여 예외관리 업무를 처리하는역할이다.
  • 배포전 성능점검
