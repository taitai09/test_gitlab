11. 성능 최적화

11.3. APP 성능 분석

애플리케이션의 성능분석정보와 데이터베이스의 SQL성능정보를 연계한 통합 성능분석화면이다. 하나의 애플리케이션은 여러 개의 DBIO로 구성되고, 하나의 DBIO는 여러 개의 SQL과 매핑된다. 즉 하나의 애플리케이션에 수행되는 모든 DBIO와 모든 SQL의 성능정보를 통해 애플리케이션에서 문제점과 병목지점을 신속하게 찾아낼 수 있다.

애플리케이션 성능분석

업무구분, 조회구분(애플리케이션, DBIO)를 선택한 후 검색하여 분석 대상 애플리케이션과 DBIO 를 조회한다. 조회된 성능정보는 아래와 같다

  • - DBIO 목록 : 검색 키 값을 LIKE 검색한 애플리케이션과 DBIO목록
  • - 애플리케이션 성능 : 애플리케이션의 일별/시간별 성능 이력과 성능 변화 추이
  • - DBIO 성능 : DBIO의 일별/시간별 성능 이력과 성능변화 추이
  • - SQL성능 : SQL의 성능이력으로 Elapsed Time, Executions 뿐만 아니라 CPU Time, Physical Read, Wait Time등의 상세 성능정보

DBIO와 SQL성능정보를 통합 분석하여 성능이슈가 단순한 데이터 증가로 인한 것인지, I/O나 Network 수준의 하드웨어 문제인지, Lock 등의 애플리케이션 구현 문제인지를 파악하는데 효율적인 활용이 가능하다.

애플리케이션 튜닝요청

애플리케이션/DBIO 목록을 선택한 후 [튜닝요청] 버튼을 클릭하면 새 탭에서 튜닝요청 화면으로 이동한다. DB명, 애플리케이션명, DBIO명, SQL Text 등의 기본정보는 자동 입력되고 추가적인 항목을 입력한 후 [튜닝요청] 버튼을 클릭하면 성능개선관리 대상으로 등록된다.
