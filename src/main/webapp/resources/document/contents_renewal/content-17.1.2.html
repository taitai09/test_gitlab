17.1.2 SQL 표준 점검

[SQL Code Inspection Result] 탭 활성화

품질점검 결과를 보기 위해서는 [SQL Code Inspection Result] 탭이 활성화 되어야 한다. Window > Show View > Other…를 클릭한다.

Show View 팝업창에서 “SQL Code Inspection Result”를 선택하고 [Open] 버튼을 클릭한다.

하단View에 [SQL Code Inspection Result] 탭이 활성화된다.

SQL 표준 점검 실행

SQL 표준 점검은 2가지 방법으로 가능하다.

[Package Explorer] 탭에서 파일명을 클릭 > 우측 마우스 클릭 > Inspect SQL Code를 클릭하거나, /p>

코드 에디터 창의 소스를 클릭하고 우측 마우스 클릭 > Inspect SQL Code를 클릭한다.

Inspect SQL Code를 클릭하면 해당 파일 내의 전체 SQL에 대한 SQL 표준 점검을 수행한다. 작업이 완료되면 “SQL code inspection completed”라는 메시지 창과 [SQL Code Inspection Result] 탭에 점검 결과가 조회된다.

결과 탭에서 행을 더블클릭하면 해당 소스의 위치로 이동한다.

  • Resource: 표준점검 대상 xml 파일명
  • ID: xml 파일 내의 SQL에 부여된 고유ID
  • Quality Inspection Index: 품질점검 결과 표준을 위배한 점검지표명
  • Description: 품질점검 결과에 대한 상세 설명
  • Path: xml 파일의 경로
품질 점검 지표 결과 예시 설명
테이블 ALIAS 미사용 EMP … ALIAS 미사용 테이블명
컬럼 ALIAS 미사용 DBID: (25,20) ALIAS 미사용 컬럼명 및 컬럼이 위치한 (행, 열)
