12. 성능 시험

12.2. DB 변경 SQL 성능 영향도 분석

개요

기존에 지원되던 힌트(HINT), 문법, 정렬(SORT) 등이 DB 업그레이드에 따라 더 이상 지원되지 않아 오류 또는 성능에 영향을 줄 수 있다. 업그레이드 이전에 영향을 받는 SQL을 검색할 수 있는 기능 및 가이드를 제공하며, 해당 SQL은 가이드에 따른 조치가 필요하다.

성능 임팩트유형
  • SELECT문 성능임팩트
    • Improved: 성능향상. TOBE BUFFER GETS < ASIS BUFFER GETS(평균)
    • Regressed: 성능저하. TOBE BUFFER GETS > ASIS BUFFER GETS(평균)
    • Unchanged: 성능동일. TOBE BUFFER GETS = ASIS BUFFER GETS(평균)
  • DML문 성능임팩트
    • Improved: FULL SCAN이 사라지거나 PARTITION ALL ACCESS가 사라진 경우
    • Regressed: FULL SCAN이 추가 발생하거나 PARTITION ALL ACCESS가 추가 발생할 경우
    • Unchanged: PLAN이 동일하거나, FULL SCAN 또는 PARTITION ALL ACCESS가 추가 발생하지 않은 경우
버퍼 임팩트(배)
  • Buffer Gets이 ASIS 대비 TOBE 성능이 향상(+) 또는 저하(-)된 비율
수행시간 임팩트(배)
  • Elapsed Time이 ASIS 대비 TOBE 성능이 향상(+) 또는 저하(-)된 비율
