11.2. SQL 튜닝대상 선정

11.2.1. 자동 선정

개요

주기적으로 튜닝 대상을 선정할 경우 자동으로 선정할 수 있다. 선정주기, 선정범위, 임계값 등의 조건을 설정하여 등록하면 자동으로 대상 SQL을 선정하고, 성능 개선 관리대상으로 자동 등록된다.

화면 구성

자동 선정 현황(선정회차)
  • 자동선정 조건에 의해 선정되어 튜닝이 진행되고 있는 TOP SQL 목록을 확인한다. 선정회차별 일괄 튜닝종료가 가능하다.

등록

  1. [초기화] 버튼을 클릭 한다.
  2. 관련 정보를 입력하고 [저장] 버튼을 클릭한다. 저장이 완료되면 아래와 같은 메시지 창이 뜬다.

수정

  1. 수정하고자 하는 행을 클릭한다.
  2. 선택된 값이 조회되며, 변경할 값을 입력하고 [저장] 버튼을 클릭한다.
  3. 메시지 창이 뜨고 데이터가 저장된다.

NOTE

통상 온라인 프로그램과 배치 프로그램의 TOP SQL 선정조건이 다르기 때문에 하나의 DB에 여러 개의 자동선정 조건을 등록하여 사용할 수 있다.

등록/수정Area > [일괄 수정] 탭

선정 조건에 대한 일괄 수정 필요시 사용한다.


전체 일괄 수정

검색된 데이터의 전체 목록을 일괄 수정한다.

  1. ‘전체’ 체크 박스를 클릭하면 하위의 모든 체크 박스가 체크된다.
  2. [일괄 수정] 탭에서 일괄 수정 대상 항목을 체크한다.
  3. 변경할 값을 입력 후 [일괄 수정] 버튼을 클릭한다.
  4. 일괄 수정하시겠습니까? 메시지가 뜬 후 YES를 클릭하면 수정이 완료된다.

개별 일괄 수정

검색된 데이터를 개별 선택하여 수정한다.

  1. 일괄 수정할 선정조건에 대한 체크 박스를 클릭한다. 멀티 선택이 가능하다.
  2. 이후의 처리 과정은 ‘> 전체 일괄 수정’ 과 처리 과정이 동일하므로 해당 매뉴얼을 참조한다.

일괄 튜닝 종료

일괄 튜닝 종료 대상 전체 선택

  1. ‘전체’ 체크 박스를 클릭하면 하위의 모든 체크 박스가 체크된다.
  2. [일괄 튜닝 종료] 버튼을 클릭한다.
  3. 일괄 튜닝 종료 팝업창이 뜨고, 튜닝종료 사유를 선택하고 내용을 입력하여 확인버튼을 클릭한다.
  4. 튜닝종료 알림 메시지창이 뜨고 튜닝이 종료된다.

일괄 튜닝 종료 대상 개별 선택

  1. 튜닝대상 SQL에 대한 체크 박스를 클릭한다. 멀티 선택이 가능하다.
  2. 이후의 처리 과정은 ‘> 일괄 튜닝 종료 대상 전체 선택’ 과 처리 과정이 동일하므로 해당 매뉴얼을 참조한다.
