10.2. 종합 진단

10.2.2 SQL 품질 진단

개요

주간 및 월간 단위로 실행된 SQL에 대한 표준을 일괄 점검한다. 등록된 스케줄러에 의해 SQL 이 수집되고, 점검을 진행하며 표준 점검 결과에 대한 집계 정보 및 표준 미준수 SQL의 상세 정보를 제공한다.

화면 구성

화면은 다음과 같은 탭으로 구성되어 있다.

Tab 명 Tab 설명
스케줄러 관리 • SQL 품질 진단을 수행할 스케줄을 등록/수정/삭제한다.
SQL 품질 진단 현황 • DB별, 스케줄러 별 품질 진단 결과에 대한 집계 정보가 조회된다.
SQL 품질 진단 결과 • SQL에 대한 품질 진단 결과를 조회한다. 점검 수행은 등록된 스케줄에 의해 자동 실행된다.
• SQL 품질 진단 수행 결과에 대한 집계 정보 및 미준수 SQL의 상세한 정보를 엑셀로 다운로드 받을 수 있다.

NOTE

품질 진단 수행 시 특정 사유로 인해 비정상 종료되더라도, 종료 시점까지 수행된 SQL들에 대해서는 진단 결과가 조회된다.

   - 전체SQL: V$를 대상으로 SQL 수집
  수집기간 • SQL을 수집할 기간
   - 주기
     일: D-1 수집
     주: D-1 ~ 이전 1 주일
     월: D-1 ~ 이전 1 개월
   - 기간: 사용자가 직접 수집 시작일자와 수집 종료일자를 입력
  Table Owner in • 특정 OWNER에 대한 수집이 필요한 경우 입력한다.
• N개의 OWNER를 입력할 경우 콤마(,)로 분리하며 대소문자는 구분하지 않으며 최대 100개까지 입력 가능하다.
   예) ORACLE, OpenPOP, orange
• 입력된 OWNER와 동일한 SQL에 대해서만 수집한다. 입력된 OWNER는 내부적으로 아래와 같은 조건으로 필터링에 포함된다.
• WHERE UPEER(OWNER) IN (‘ORACLE’, ‘OPENPOP’, ORANGE’)
  Module like • 소스 DB에서 특정 MODULE에 대한 수집이 필요한 경우 입력한다.
• N개의 MODULE을 입력할 경우 콤마(,)로 분리하며 대소문자는 구분하지 않으며 최대 100개까지 입력 가능하다.
   예) JDBC, dbsnmp • 입력된 MODULE이 포함된 SQL에 대해서만 수집한다. 입력된 MODULE은 내부적으로 아래와 같은 조건으로 필터링에 포함된다.
   WHERE (UPEER(MODULE) LIKE ‘%JDBC%’ OR
• UPEER(MODULE) LIKE ‘%DNSMP%’)
  Filter SQL

FilterSQL

   • [Filter SQL] 버튼을 클릭한다.

   • Filter SQL 팝업 창이 뜨고, Module, Action, Parsing Schema Name, SQL_TEXT, SQL ID 항목에 대한 SQL을 추가한 후 [저장]버튼을 클릭한다.

   • 저장된 내용이 [Filter SQL] 버튼 옆에 보여진다.

   • SQL_TEXT 선택 시 LIKE , NOT LIKE 만 선택 가능하다.
