10.3.1. SQL 진단

플랜 변경, 악성 신규 SQL, Temporary 과다 사용에 의한 Swaping 발생 등 DB성능 장애를 유발할 수 있는 다양한 유형의 SQL을 매일 정기적으로 진단하며 진단 대상은 아래와 같다.

  • - 플랜변경 SQL: 플랜이 변경되고 성능이 설정한 임계값 이상 저하된 SQL
  • - 신규 SQL : 신규 발생한 SQL로 DB activity가 설정한 임계값 이상인 SQL
  • - Temp 과다사용 SQL : Temporary Tablespace 를 설정한 임계값 이상 사용하는 SQL
  • - Literal SQL : 바인드 처리하지 않는 Literal SQL
  • - Fullscan SQL : Table Full Scan 또는 Index Full Scan하면서 성능이 설정한 임계값 이상인 SQL
  • - 조건절 없는 Delete : 조건절 없이 수행된 Delete 문으로 사용자의 실수인지 또는 Truncate변경 가능한지 검토
  • - TOPSQL : Execution, Buffer Gets, Elapsed Time 이 설정한 임계값을 초과하는 SQL
  • - OFFLOAD 비효율 SQL : FULL SCAN을 동반하면서 Offload 효율이 설정한 임계값 이하인 SQL
  • - OFFLOAD 효율저하 SQL : 일주일전 평균 Offload 효율보다 설정한 임계값 이상 효율이 저하된 SQL
