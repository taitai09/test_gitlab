13.3. SPACE 분석

13.3.2. Reorg 대상 분석

운영중인 DB에 계속적인 Insert / Update / Delete 발생으로 인한 Fragmentation 발생하면, Chained Row가 발생하여 Disk I/O가 증가되며, 결과적으로 서비스 성능이 저하된다. 조각난 데이터 및 인덱스를 조각나지 않은 연속된 페이지로 재 빌드함으로써 복잡하고 불필요하게 공간을 차지하고 있는 테이블 및 인덱스를 최적화 할 수 있으며, 이로 인해 액세스 되는 블록도 감소시켜 SQL쿼리의 성능을 향상시킬 수 있다.

Reorg 대상분석

테이블의 DML 분석정보를 토대로 TOP DML 테이블에 대한 DML발생추이와 Reorg 효과를 확인하여 Reorg 작업대상을 식별하는데 효과적으로 사용한다.

Reorg 대상분석화면은 세 개의 카테고리로 구성되었다

  • - TOP DML Table : DML이 많이 발생한 TOP 10 테이블 현황
  • - TOP DML Table History : DML이 많이 발생한 TOP 10 테이블의 DML발생추이
  • - Table List : 전체 테이블의 목록과 Reorg에 의해 절감되는 사이즈 등의 Reorg 분석정보
